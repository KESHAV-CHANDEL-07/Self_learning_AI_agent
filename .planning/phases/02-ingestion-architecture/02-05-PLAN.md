# Plan 2-5: Config Extensions & Daemon Integration

## Objective
Extend the existing configuration system to support `ignore_patterns` and `plugins` sections in `~/.sg_agent/config.yaml`. Wire the config into the daemon startup so that user-specified ignore patterns are passed to the `PathFilter` and the plugins directory is configurable. Update the `run_daemon_process()` function in `cli.py` to pass config through to the daemon.

## Code Patterns (Model Resilience)
Reference these existing files for code style:
- `agent/config.py` — This IS the file being modified.
- `agent/cli.py` line 18-27 (`run_daemon_process`) — This is the daemon startup path.

Conventions:
- Config keys: `snake_case` (e.g., `ignore_patterns`, `plugins_dir`).
- Error handling: Missing config keys → use defaults. Bad values → log warning, use defaults.

## Must-Haves
- All new config keys have sensible defaults in `DEFAULT_CONFIG`.
- Missing config keys never crash — defaults are always used.
- Config-specified `ignore_patterns` are passed through to `PathFilter`.
- Config-specified `plugins_dir` is passed through to `PluginRegistry`.
