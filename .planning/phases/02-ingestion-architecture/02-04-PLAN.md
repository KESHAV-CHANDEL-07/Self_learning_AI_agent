# Plan 2-4: Graceful Shutdown & Error Handling

## Objective
Upgrade the existing daemon to support a 10-second graceful shutdown timeout, integrate the `PathFilter` for event filtering, integrate the `PluginRegistry` for event dispatch, and integrate the `RepoManager` for workspace resolution. Add robust error handling with clear status reporting for all error types (permission, network, invalid paths).

## Code Patterns (Model Resilience)
Reference these existing files for code style:
- `agent/daemon.py` — This IS the file being modified. Match existing structure exactly.
- `agent/cli.py` — CLI commands being extended. Match existing Typer patterns.

Conventions:
- Error handling: Catch specific exceptions (`PermissionError`, `git.exc.GitCommandError`) before generic `Exception`.
- Logging: Continue using `logging.getLogger("agent.daemon")`.
- Shutdown: Use `threading.Event` for cleaner shutdown signaling vs boolean flag.

<task type="auto">
  <name>Upgrade DaemonWatcher with graceful shutdown timeout</name>
  <files>agent/daemon.py</files>
  <action>
    1. Add imports: `import threading` at top of file.
    2. Add import: `from agent.path_filter import PathFilter`.
    3. Add import: `from agent.plugin_registry import PluginRegistry`.
    4. Add constant: `SHUTDOWN_TIMEOUT = 10`.
    5. Modify `WorkspaceHandler.__init__`:
       - Accept `path_filter: PathFilter` and `plugin_registry: PluginRegistry` parameters.
       - Store as `self.path_filter` and `self.plugin_registry`.
    6. Modify `WorkspaceHandler.on_any_event`:
       - Replace the hardcoded `.git` check with: `if event.is_directory or self.path_filter.should_ignore(event.src_path): return`.
       - After logging the event, create an event dict: `event_data = {"type": event.event_type, "path": event.src_path, "is_directory": event.is_directory}`.
       - Create context dict: `context = {"workspace": self.path_filter}`.
       - Call `self.plugin_registry.run_all(event_data, context)`.
       - Wrap the plugin call in `try/except Exception` to prevent plugin errors from killing the handler.
    7. Modify `DaemonWatcher.__init__`:
       - Accept optional `path_filter: PathFilter = None` and `plugin_registry: PluginRegistry = None`.
       - Store them, using defaults if None: `self.path_filter = path_filter or PathFilter()`, `self.plugin_registry = plugin_registry or PluginRegistry()`.
       - Replace `self.running = False` (boolean) with `self._shutdown_event = threading.Event()`.
       - Add property `running` that returns `not self._shutdown_event.is_set()`.
    8. Modify `DaemonWatcher.start`:
       - Replace `self.running = True` with `self._shutdown_event.clear()`.
       - Replace `while self.running` with `while not self._shutdown_event.is_set()`.
       - Pass `self.path_filter` and `self.plugin_registry` to `WorkspaceHandler(...)`.
       - Call `self.plugin_registry.discover()` before starting the observer.
    9. Modify `DaemonWatcher.shutdown`:
       - Call `self._shutdown_event.set()`.
       - If `self.observer`: `self.observer.stop()`, then `self.observer.join(timeout=SHUTDOWN_TIMEOUT)`.
       - If observer is still alive after join: `logger.warning("Observer did not stop within timeout, forcing exit")`.
    10. Add a list `self._errors: list = []` to `DaemonWatcher.__init__` to track skipped errors.
    11. Add method `get_errors(self) -> list` to return the error list.

    Match style from: `agent/daemon.py` (existing structure)
  </action>
  <verify>python -c "from agent.daemon import DaemonWatcher; print('Import OK')"</verify>
  <done>DaemonWatcher supports 10s graceful shutdown, integrates PathFilter and PluginRegistry, and tracks errors.</done>
</task>
