# Plan 1-3: Daemon Core

## Objective
Implement a continuous background listener using `watchdog` to monitor the target workspace. The daemon must gracefully handle crashes by restarting up to 3 times with exponential backoff before exiting.

## Code Patterns (Model Resilience)
Reference these existing files for code style:
- `agent/config.py` — Consistency in typing and imports.

Conventions:
- File naming: `snake_case`.
- Error handling: Catch broad exceptions within the daemon loop, increment crash counter, and sleep.
- Types: Use type hints for all functions, especially the signal handlers.

## Tasks

<task type="auto">
  <name>Implement Watchdog Observer Loop</name>
  <files>agent/daemon.py, tests/test_daemon.py</files>
  <action>
    1. Import `time`, `signal`, `logging`, and `Observer`, `FileSystemEventHandler` from `watchdog`.
    2. Define a class `WorkspaceHandler(FileSystemEventHandler)` that simply logs valid file events (omit directories and ignored patterns like `.git`).
    3. Create a `DaemonWatcher` class that manages an `Observer`.
    4. Implement dynamic restarting logic: if the observer or event handler raises an exception, catch it, log the error, and increment a `crash_count`.
    5. If `crash_count &lt; 3`, `time.sleep(2 ** crash_count * 5)` and restart the loop. If it hits 3, exit cleanly.
    6. Implement a `shutdown` method triggered by `SIGTERM` and `SIGINT` signals using the `signal` module to stop the `Observer` safely and flush remaining tasks.
    7. Write unit tests utilizing a mocked observer to verify the restart logic and exit sequence.
  </action>
  <verify>pytest tests/test_daemon.py</verify>
  <done>The daemon loop handles crashes cleanly, restarts up to 3 times, and shuts down safely on SIGINT/SIGTERM.</done>
</task>

## Must-Haves
- 5s default poll interval (if polling is manually required, otherwise use standard observer).
- 3 auto-restarts with exponential backoff on crash.
- Clean shutdown on `SIGTERM` or `SIGINT`.
