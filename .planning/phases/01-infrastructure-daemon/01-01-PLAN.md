# Plan 1-1: Configuration System

## Objective
Establish a robust configuration management system using PyYAML. This system will define sensible defaults, allow a global configuration at `~/.sg_agent/config.yaml`, and enable a project-specific override via a local `.sg_agent.yaml` file.

## Code Patterns (Model Resilience)
Reference these existing files for code style:
- `agent/utils/` — Match the function definitions and standard docstrings.

Conventions:
- File naming: `snake_case` for Python (`config.py`).
- Error handling: Log a warning and fallback to defaults if a file is malformed; raise explicit `ConfigError` only for unrecoverable issues.
- Types: Use standard Python `typing` (`Dict`, `Any`, `Path` from `pathlib`).

## Tasks

<task type="auto">
  <name>Implement Config Loader</name>
  <files>agent/config.py, tests/test_config.py</files>
  <action>
    1. Import `yaml`, `Path` from `pathlib`, and `logging`.
    2. Define a base dictionary `DEFAULT_CONFIG` containing default settings (e.g., `log_level: "INFO"`, `poll_interval: 5`).
    3. Create a `load_config(global_path: Path, local_path: Path) -> dict` function.
    4. Inside `load_config`, initialize the config with `DEFAULT_CONFIG`.
    5. Check if `global_path` exists. If so, read it with `yaml.safe_load()` and update the config dictionary.
    6. Check if `local_path` exists. If so, read it with `yaml.safe_load()` and update the config, overriding previous settings.
    7. Wrap `yaml.safe_load` in a `try/except` block catching `yaml.YAMLError`; if an error occurs, fallback to defaults and log a warning.
    8. Write unit tests in `tests/test_config.py` validating the load hierarchy (defaults -> global -> local).
  </action>
  <verify>pytest tests/test_config.py</verify>
  <done>The `load_config` function reliably returns a merged configuration dictionary and all tests pass.</done>
</task>

## Must-Haves
- `yaml.safe_load` must be used for security.
- Local configurations must override global configurations.
- Missing files must not crash the application; they should smoothly fall back to defaults.
