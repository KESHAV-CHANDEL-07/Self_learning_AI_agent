# Plan 1-2: Structured Logging

## Objective
Implement structured, size-rotated text logging for the agent to ensure all decisions and errors are recorded sequentially without filling up the disk. The logs will be written to `~/.sg_agent/logs/agent.log`.

## Code Patterns (Model Resilience)
Reference these existing files for code style:
- `agent/config.py` — Adopt the same import style and typing standards.

Conventions:
- File naming: `snake_case`.
- Error handling: If the log directory cannot be created, raise a clear `PermissionError`.
- Types: Use standard Python `logging`.

## Tasks

<task type="auto">
  <name>Setup Rotating File Logger</name>
  <files>agent/logger.py, tests/test_logger.py</files>
  <action>
    1. Import `logging`, `logging.handlers.RotatingFileHandler`, and `Path` from `pathlib`.
    2. Create a function `setup_logger(log_dir: Path, log_level: str = "INFO") -> logging.Logger`.
    3. Ensure `log_dir` exists; if not, create it recursively using `log_dir.mkdir(parents=True, exist_ok=True)`.
    4. Configure a `RotatingFileHandler` pointing to `log_dir / "agent.log"` with `maxBytes=10*1024*1024` (10MB) and `backupCount=5`.
    5. Set the formatter to standard text format: `%(asctime)s - %(name)s - %(levelname)s - %(message)s`.
    6. Attach the handler to the root logger and set the base level according to `log_level`.
    7. Write tests validating that directory creation works and logging to file succeeds.
  </action>
  <verify>pytest tests/test_logger.py</verify>
  <done>A reusable logger object is properly configured to rotate at 10MB and keep 5 backups.</done>
</task>

## Must-Haves
- Must enforce max file size of 10MB and keep maximum 5 files.
- Must ensure the destination directory is created automatically if it doesn't exist.
- Standard text format must include timestamps.
